# Launch distribution checklist (Bangalore-first)

**Labels:** `growth`, `priority: high`  
**Milestone:** Distribution

## Problem
The app is useless without users and dense data. Code is ~40% of "useful" — distribution is the other 60%.

## Week 1
- [ ] Deploy live URL
- [ ] Manually add every hackathon/meetup you know about this month
- [ ] Share link in your college WhatsApp / Telegram groups
- [ ] Post on college Instagram meme / tech page

## Week 2
- [ ] Recruit 2 campus scouts (see ambassador issue)
- [ ] Reach out to one tech club coordinator per college
- [ ] Run `npm run scrape` daily — confirm feed isn't empty

## Week 3
- [ ] Create Instagram: "This week's Bangalore hackathons" weekly post
- [ ] Get mentioned in one Bangalore tech Telegram/Discord (GDG, PyData)
- [ ] Moderate all submissions within 24 hours

## Week 4
- [ ] 20+ attendable events in feed at all times
- [ ] Cross-post to student newsletters if possible
- [ ] Collect feedback: what's missing?

## Acceptance criteria
- ≥50 unique visitors in first month (check Vercel analytics)
- ≥5 crowdsourced submissions total
- Feed never empty for more than 48 hours

## Do not expand to other cities until Bangalore feed is dense and reliable
